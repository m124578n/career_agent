# 設計規格：溫暖指揮艙（Warm Cockpit）前端重新設計

- 日期：2026-06-27
- 範圍：前端 UI/UX 重新設計
- 狀態：設計已確認，待寫實作計畫

## 背景與目標

`career_agent` 前端（job-tracker）是一個 AI 求職工具：履歷診斷 →
職缺契合度分析 → 求職信生成 → 追蹤清單。技術為 React 18 + Mantine 7 +
Vite，現有視覺是「指揮艙／終端機」風格（深色 ink 底、tangerine/teal 雙訊號色、
大量等寬字、`[+]`/`[!]` 與 `//` 終端符號）。

現況評估（設計師視角）：工藝與工程紮實（無障礙、跨頁狀態、狀態處理都到位），
但**語氣偏冷硬、工具感重，對「求職者的情緒與親和力」適配不足**。本次目標是
**保留品牌 DNA、補上親和力**，讓它從「精準的儀器」變成「冷靜可靠的副駕」。

非目標（Out of scope）：
- 不改後端、API、資料流或業務邏輯。
- 不更換元件庫（續用 Mantine + theme）。
- 不打掉重練——既有的無障礙、狀態持久化、輪詢等工程一律保留。

## 設計原則（拍板用的單一準則）

> **「像一位冷靜可靠的副駕」——資料的地方保持精準，對人說話的地方保持溫暖。**

- **精準**（等寬字、分數、終端感）：只留給**資料**。
- **溫暖**（口語、柔色、留白、鼓勵）：給**所有對使用者說話的時刻**。

這條線解決「warm × cockpit」的張力，避免做成不冷不暖的半套。

## 工作方式與範圍

1. 先做**共用設計基底**（token、字體規則、元件與文案語言）。
2. 套用到**一頁示範**：`履歷與目標`（ResumeSetup，預設落地頁）。
3. 跑起 app 截圖確認方向，OK 後再**一頁頁**推到其餘頁面。

基底採**加法式擴充、盡量沿用既有 `--jt-*` 變數名**：共用面板/卡片在尚未改版的
頁面會「順便變溫暖」但不致破版；完整改版仍逐頁進行。共用 class 盡量只動 token，
視覺差異由 theme/token 控制，降低 rollout 中途的不一致觀感。

## 設計基底

### 1. 色彩：語意分流 + 把溫度做在看得見的地方

**語意分流（修正現況的色彩超載）**：
- `tangerine #ff6a3d`：**只當主要行動**（CTA、主按鈕）。
- `teal #34d6c8`：**正向 / 契合 / 連結**。
- **新增 `amber/gold`（暫定 `#e9a23b` 一帶，實作時校對對比）：「可加強的地方」**，
  與主行動色徹底分開，避免「警告 vs 要我點」的混淆。
- admin/全站等次要標記改用中性或既有色，不再與語意色搶位。

**溫度做在看得見處**（而非面板基底差幾個 hex 的無感調整）：
- 頂部暈光、淡色襯底、邊框**微帶暖**；情緒時刻讓 tangerine 多露臉。
- 面板基底可中性偏暖極微調，但不依賴它產生「暖感」。

**層次（修正過於平面）**：
- 建立明確表面層次：`底 → 面板 → 浮起面板`，用**柔和陰影 + 低對比邊框**
  取代到處生硬的 1px 線，讓主內容與外框一眼可分。
- 狀態用**淡色底襯**：亮點＝teal 6–10% 底、可加強＝amber 6–10% 底。

### 2. 字體：等寬字只留給「該精密」的地方

- 標題續用 **Space Grotesk**、內文 **IBM Plex Sans**。
- **IBM Plex Mono 只用於：分數、數字、標籤值**；section 小標與說明文字改一般字體。
- 大幅減少「英文大寫 + 寬字距」eyebrow，改**口語句式小標**；mono 僅極少數點綴。

### 3. 間距與形狀

- 圓角放大：預設 sm→md；面板 10→14/16px。加大內距與垂直節奏、放大可點區域。
- 保留「淡網格 + 頂部暈光」氛圍，但更克制、暈光更暖。

### 4. 動態（輕柔、尊重偏好）

- 結果/卡片淡入、卡片 hover 微浮起、按鈕按壓回饋——皆輕量。
- 保留既有 `@media (prefers-reduced-motion: reduce)` 關閉機制。

### 5. Token 單一來源（修正兩個真相來源）

- 現況 `theme.ts` 的 ink 色階與 `global.css` 的 `--jt-*` 內容重複，易漂移。
- 本次定義**單一來源 + 對應表**：以 CSS 變數為呈現層真相，theme.ts 透過變數引用
  （或反之，實作時擇一並註記），避免日後不同步。

## 元件與文案語言

### 元件
- **面板 → 柔和卡片**：標題列為「友善標題 +（可選）小 mono tag」，去除 `// INPUT` 大寫。
- **標記符號**：`[+]`/`[!]` → **柔和圓點/小圖示 chip + 淡色底**
  （teal 點＝亮點、amber 點＝可加強），不再用方括號。
- **分數**：冷數字**配一句白話定性**，例 `87 · 很適合`，把評判感包進鼓勵語氣
  （主要影響職缺頁，但語言在基底先定調）。
- **按鈕**：tangerine 主行動更柔、主次階層清楚、尺寸一致。
- **空 / 載入 / 錯誤狀態**：一律「圖示 + 一句鼓勵 + 明確下一步」。

### 情緒處理（friendly 的本質）
- **先講亮點、再講可加強**；可加強用**成長口吻**，必要時限制顯示數量。
- 旅程感：完成一步給**下一步出口**（如：診斷完成 → 「去找職缺」CTA），
  串起 履歷→職缺→追蹤。

### 文案改寫（樣例）
| 現況 | 改為 |
|---|---|
| `輸入 // INPUT` | 上傳與設定 |
| `診斷讀數 // READOUT` | 你的診斷結果 |
| `優勢 // STRENGTHS` | 你的亮點 |
| `待補強 // GAPS` | 可以加強的地方 |
| `等待輸入 // 上傳履歷並設定目標後執行診斷` | 還沒有診斷結果　上傳履歷、填好目標職位，我就幫你看看亮點和可以加強的地方。 |
| `診斷失敗 // 請稍後再試或確認後端設定` | 分析沒成功，先確認網路或稍後再試一次。 |

## 示範頁：履歷與目標（ResumeSetup）

套用以上全部：
- **頁首**：溫暖標題 + 一句親切說明。
- **左卡「上傳與設定」**：友善拖放上傳區，四態皆有圖示與清楚文字
  （空 / 解析中 / 已載入 / 失敗）；輸入欄加 helper 提示；醒目「執行診斷」CTA + 小提示。
- **右卡「你的診斷結果」**：親切空狀態、溫暖的分析中動畫；結果分「亮點 / 可以加強」
  兩區，柔色底列 + 圓點標記（無方括號），數量做成友善小 chip；
  完成後提供「去找職缺」下一步出口。

## 實作落點

- **基底**：
  - `frontend/src/theme.ts`：圓角、色（含新增 amber 語意）、層次 token。
  - `frontend/src/styles/global.css`：`--jt-*` 變數加溫、`.jt-*` 工具類調整、
    陰影/層次、token 單一來源對應。
- **示範頁**：`frontend/src/pages/ResumeSetup.tsx`。
- **抽出可重用小元件（供後續頁面共用）**：診斷項目列（亮點/可加強）、
  友善空狀態元件。

## 無障礙約束（須維持，不可退步）

- 所有文字與狀態色維持 ≥ 4.5:1（含 amber 與淡底襯上的文字，實作時校對）。
- 保留 `:focus-visible` 焦點環、icon 按鈕 `aria-label`、`prefers-reduced-motion`。

## 驗證計畫

- `npm run dev` 跑起 app，實際截圖檢視示範頁。
- 確認左卡四態（空/解析中/已載入/失敗）與右卡三態（空/分析中/結果）視覺正確。
- 抽查關鍵對比值達 AA。
- 確認其餘頁面未破版（加法式擴充的副作用檢查）。

## 後續 rollout（示範頁確認後）

依序套用基底到：登入頁 → 職缺契合度 → 追蹤清單 → 關於我 → 側邊導覽，
每頁沿用本 spec 的基底與語言，逐頁跑起確認。
